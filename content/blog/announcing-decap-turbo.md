Decap Turbo is a cloud-based upgrade for teams already using Decap CMS in production. It is built to remove the operational overhead around auth and permissions, while making the CMS experience noticeably faster for editors.

## Decap CMS stays open source

Decap CMS remains the same open source, self-installed Git-based CMS the community relies on. We will keep maintaining and improving it as the core of our offering.

Decap Turbo is an optional upgrade layer for teams that want:

- Managed authentication
- User roles and permissions
- Better database proxy performance for faster CMS loading and caching.

If you are happy with your current Decap CMS setup, nothing changes.

## Why we built Turbo

As Decap projects become more complex, these are the common requests from larger teams:

- "We need simpler authentication management across environments"
- "We need stricter role-based access for different content teams"
- "We need better performance when we have a large content repository"

Decap Turbo is our answer to those production needs.

## What you get with Decap Turbo

### 1. Authentication handling without the headaches

Turbo provides managed authentication so teams can stop maintaining custom auth glue across providers and environments.

### 2. Roles and permissions built for real teams

Define who can access which collection, so writers, editors, and admins each get the right level of control.

### 3. A faster CMS experience

The APIs of Git providers are not built for high-performance CMS workflows. Turbo adds a database proxy layer to improve performance and caching. With Turbo, you can now have thousands of content items without the CMS slowing down.
